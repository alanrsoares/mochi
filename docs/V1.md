# mochi v1 — scope

**Thesis: v1 is a self-hosting showcase.** The claim v1 makes: *mochi compiles mochi* —
a shipped `mochic` binary, built by itself, byte-stable at the fixpoint, accepting the
full surface syntax, with error messages good enough to show off. The bootstrap compiler
is the product; everything in scope serves that claim's honesty and polish.

## In scope

### 1. Full self-host parity (surface syntax)

The self-hosting claim is honest only if `bootstrap/parser.mochi` accepts everything
`src/parser.ts` accepts.

- [ ] Port record update `{ ...base, f: v }` (ADR 0021) to `bootstrap/parser.mochi`.
- [ ] Port or-patterns `| A | B => body` (ADR 0022) to `bootstrap/parser.mochi`.
- [ ] Dogfood both in bootstrap sources where they pay (record update was motivated by
      `infer.mochi`/`check.mochi` threading `St`/`TSt` field-by-field — use it there).
- [ ] **Parity gate, ongoing rule:** new surface syntax lands in `src/` and `bootstrap/`
      in the same change, with differential coverage. No new parity debt.

### 2. Module-graph self-hosting

Today `mochic` compiles import-bearing modules in **open-world** mode: imports are
emitted unresolved, so the self-hoster has no cross-module inference or exhaustiveness —
it is weaker than the TS compiler it claims to replace. The biggest v1 work item:

- [ ] Port the `module.ts` driver (graph resolve, DFS topo-sort, cycle detection,
      export-scheme threading, imported-registry exhaustiveness) to the bootstrap.
- [ ] Host IO stays at the `host.js` seam (sync-return `Result`, per the no-
      `Promise<Result>` rule).
- [ ] Fixpoint upgraded: `scripts/fixpoint.ts` runs the **graph** build through the
      self-hosted driver, stage2 ≡ stage3 byte-for-byte.

### 3. String interpolation (ADR 0023)

The one surface addition v1 admits — highest-leverage feature by dogfood evidence
(123 `cat([...])` sites in bootstrap; 83 in codegen.mochi alone). Serves both pillars:
error-message ergonomics + bootstrap readability. Codegen = native JS backtick
template literal.

- [ ] Core: lexer `${}` mode, `interp` node, holes unify `string`, template-literal
      codegen, formatter round-trip.
- [ ] Bootstrap parity (lexer.mochi + parser.mochi) per the parity gate.
- [ ] Dogfood sweep: migrate `cat([...])` sites where interpolation reads better.

### 4. Error-message quality (already started)

The axis chosen for the v1 push (first ship: alias folding in unify errors, de42dcd).
Continue while items stay cheap; errors are the showcase's face.

- [ ] Sweep remaining error sites for span coverage + actionable hints
      (candidates surfaced case-by-case; each ships with a guard test).
- [ ] Bootstrap error parity stays pinned by the differential suites
      (message + span, as today).

### 5. Docs

- [ ] **Language tour** — every construct, runnable example each. "Learn mochi in
      20 minutes."
- [ ] **Prelude/stdlib reference** — every namespace member with its signature.
      Generate from the `prelude.ts` tables (one source of truth, like
      `gen-prelude.ts`).
- [ ] **Getting started** — install, compile, editor/LSP setup, first project.

## Out of scope (deliberate)

| Item | Why out |
| --- | --- |
| Traits (ADR 0018 prototype) | Headline feature, own epic; bridge decision (structural `eq`/`compare` + `-By` combinators) covers v1. v2 material. |
| `>>=` / custom operators | Not scoped; collides with no-overloading. Needs the traits outcome first. |
| Quadratic-subst perf (PATH §3) | Only when it hurts. Compile times are single-digit ms. |
| Parallel compile | Deliberately deferred (roadmap); needs explicit go-ahead. |
| Rebrand (REBRAND.md) | Separate track; naming churn mid-push costs parity noise. |
| JSX (docs/jsx.md) | Design note only; not part of the self-hosting claim. |
| New surface syntax beyond ADR 0021/0022/0023 | Surface frozen for v1 — parity gate makes every addition cost double. |

## Definition of done

1. `bun run fixpoint` green: stage2 ≡ stage3 byte-for-byte across the full module
   corpus **through the self-hosted graph driver**.
2. Zero parity gaps: every construct `src/parser.ts` accepts, `bootstrap/parser.mochi`
   accepts, pinned by the differential suites.
3. `bun run check` green (as always).
4. Tour + stdlib reference + getting started exist and are current.
5. A fresh clone can: install → build `mochic` with itself → compile a small
   multi-module project → get a good error when they break it.
